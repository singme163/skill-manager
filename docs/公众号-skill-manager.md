## 正文

如果你同时在用 Claude Code 和 Codex，大概率遇到过和我一样的问题。

Skill（技能）是现在 AI 编程工具最好用的扩展方式：一个文件夹、一份 `SKILL.md`，就能教会 AI 一套新本事——生成 PPT、修图、分析磁盘空间……社区里的优质 skill 也越来越多。

但管理它们的体验，还停留在石器时代：

- 装了哪些 skill？得 `ls ~/.claude/skills` 和 `ls ~/.codex/skills` 各看一遍
- Claude Code 里那个好用的 skill，想在 Codex 里也用？手动 `cp -r`
- 网上看到一个不错的 skill 仓库？clone 下来、找到目录、再拷进去
- 每个 skill 是干嘛的？挨个打开 SKILL.md 看

说白了，**skill 生态在飞速发展，但「管理 skill」这件事没人管**。

于是我做了 Skill Manager——一个 macOS 原生 App，把两个工具的 skill 放进同一个窗口里。

【截图：App 主界面三栏视图】

### 一眼看清两边装了什么

打开 App，左边按来源筛选（全部 / Claude Code / Codex），中间是 skill 列表：名称、用途描述、体积、更新时间，一目了然。

几个小细节：

- **同一个 skill 两边都装了？** 列表里合并成一条，挂上双徽标，不用来回对
- **顶部搜索框**，按名称和描述即时过滤，skill 装多了也不怕
- **目录变化自动刷新**——你在终端里手动装了个 skill，回到 App 它已经出现在列表里了

### 四种方式装 skill

点右上角「+」：

1. **导入本地文件夹 / zip**——直接把文件夹拖进窗口也行
2. **新建空白 skill**——填个名称和描述，自动生成符合规范的 SKILL.md 骨架，马上开写
3. **跨工具复制**——Claude Code 里的 skill 一键复制到 Codex，反过来也行，这是我用得最多的功能
4. **从 GitHub 链接安装**——粘贴仓库地址（支持 `/tree/main/子目录` 这种深链），自动下载、找到里面所有 skill、勾选安装

【截图：GitHub 安装 / 导入弹窗】

所有安装都可以选目标：装到 Claude Code、装到 Codex，或者两个一起。遇到同名冲突会先问你，确认覆盖的话，旧版本也是先进废纸篓而不是直接没了。

### 不只是看，还能改

选中一个 skill，右侧详情页有四个标签：

- **预览**：SKILL.md 渲染成排版好的文档
- **编辑**：内置编辑器直接改，⌘S 保存，保存前还会帮你校验 frontmatter 格式，避免改坏了 AI 认不出来
- **文件**：这个 skill 带了哪些脚本、参考文档、资源，清单列出来
- **用法**：自动生成这个 skill 的调用方式——斜杠命令、自然语言提示词模板，点一下就复制，发给 AI 就能用

【截图：详情页「用法」标签】

「用法」这个标签是我自己最喜欢的：装了 skill 却忘了怎么触发，是真实发生过很多次的事。

### 删除？放心删

所有删除操作都是**移入系统废纸篓**，永不硬删。双端都装了的 skill，删除时可以选只删一边还是两边一起。手滑了去废纸篓捞回来就行。

### 一些你可能关心的

- **原生 SwiftUI**，不是 Electron 套壳。整个 App 就几 MB，秒开，自动适配深色模式
- **零网络依赖**（除了 GitHub 安装功能），你的 skill 数据不会离开本机
- skills 目录路径可以在设置里自定义，非标准安装位置也能用

### 彩蛋：这个 App 是怎么来的

坦白说：这个 App 从 PRD 到代码到单元测试，是我和 Claude Code 结对完成的——先让它输出一版 PRD，我确认了技术选型（SwiftUI 原生）和功能边界，然后它搭工程、写代码、写测试、打包，一个下午出了能用的版本。

更有意思的是，我这台 Mac 上**连 Xcode 都没装**，只有 Command Line Tools。整个项目是纯 Swift Package Manager 工程，`swift build` 编译，一个脚本打包成 .app。

用 AI 编程工具，做了一个管理 AI 编程工具的工具。这大概就是 2026 年写代码的样子。
